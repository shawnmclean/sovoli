# Documentation Guide

This document explains the structure and organization of documentation in the Sovoli project.

## Documentation Structure

The `docs/` directory contains organized documentation for different aspects of the project:


### Data Layer Documentation

**Location**: `docs/data/`

Documentation related to the data layer, CRM, analytics, metrics, and data structure:

- `crm.md` - CRM system documentation
- `onboarding.md` - Tenant onboarding guide (step-by-step process for adding new organizations)
- `guyana-schools-table.md` - Data structure documentation

This section covers:
- How to access analytics
- How metrics work
- Data layer structure and organization
- Database schemas and relationships

### Product Documentation

**Location**: `docs/product/`

Product-related documentation including marketing, user personas, market tests, and funnel designs:

- `user-archetypes.md` - User persona definitions

This section covers:
- Product marketing strategies
- User personas and archetypes
- Market-level tests and experiments
- Funnel designs and user journeys

## How to Navigate Documentation

1. **For data layer questions**: Refer to `docs/data/` for CRM, analytics, and data structure
2. **For product context**: See `docs/product/` for user personas and product strategy
3. **For development setup**: See root `AGENTS.md` for technical setup and workflows

## Contributing to Documentation

- Document data layer changes in `docs/data/`
- Update product documentation when personas or strategies change
- Follow existing documentation patterns and structure
